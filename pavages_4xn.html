Ce problème est résolu de manière progressive, et c'est l'occasion d'utiliser :

Problème simplifié

  • Si on ne pave qu'avec des barres verticales, il n'y a qu'une seule façon de paver un rectangle 4×n4\times n4×n.
    • ∀n∈Nan=1\forall n \in \mathbb N \quad a_n = 1∀n∈Nan​=1
  • Si on ne pave qu'avec des barres verticales et des carrés, il est facile de voir que (an)n(a_n)_n(an​)n​ correspond à la suite de Fibonacci décalée, à savoir :
    • a0=a1=1a_0 = a_1 = 1a0​=a1​=1, et ∀n∈Nan+2=an+1+an\forall n \in \mathbb N \quad a_{n+2} = a_{n+1} + a_n∀n∈Nan+2​=an+1​+an​

Relation d'ordre 888 pour ana_nan​

D'après OEIS®

Sur On-Line Encyclopedia of Integer Sequences, cette suite est référencée sous A220123. On y découvre une autre relation de récurrence :

Pour n⩾8n\geqslant8n⩾8, on a : −an−8+an−6−an−5+5an−4+an−2+an−1=an-a_{n-8}+a_{n-6}-a_{n-5}+5a_{n-4}+a_{n-2}+a_{n-1} = a_{n}−an−8​+an−6​−an−5​+5an−4​+an−2​+an−1​=an​

On peut vérifier cette relation sur nos valeurs :

for n in range(8, 24):
    assert  -a[n-8] +a[n-6] -a[n-5] +5*a[n-4] +a[n-2] +a[n-1] == a[n], f"Échec avec n = {n}"

Remarque : Pour une suite qui ne serait pas référencée, l' algorithme de Berlekamp–Massey (en) permet de déterminer une relation de récurrence en fonction des premiers termes.

Preuve de cette relation de récurrence

Pour n⩾8n\geqslant 8n⩾8, on a :

  • an=an−1+an−4+an−2+2bn−4+cn−4(I)a_n = a_{n-1} + a_{n-4} + a_{n-2} + 2b_{n-4} + c_{n-4} \quad (I)an​=an−1​+an−4​+an−2​+2bn−4​+cn−4​(I)
  • bn−4=bn−6+an−4=bn−8+an−6+an−4b_{n-4} = b_{n-6} + a_{n-4} = b_{n-8} + a_{n-6} + a_{n-4}bn−4​=bn−6​+an−4​=bn−8​+an−6​+an−4​
  • cn−4=cn−8+an−4c_{n-4} = c_{n-8} + a_{n-4}cn−4​=cn−8​+an−4​

On déduit :

  • an=an−1+an−4+an−2+2bn−8+2an−6+2an−4+cn−8+an−4(II)a_n = a_{n-1} + a_{n-4} + a_{n-2} + 2b_{n-8}+ 2a_{n-6} +2a_{n-4}+ c_{n-8} + a_{n-4}\quad (II)an​=an−1​+an−4​+an−2​+2bn−8​+2an−6​+2an−4​+cn−8​+an−4​(II)

Or, d'après (I)(I)(I) (valable aussi pour n⩾4n\geqslant 4n⩾4), on a pour n⩾8n\geqslant 8n⩾8 :

  • an−4=an−5+an−8+an−6+2bn−8+cn−8(III)a_{n-4} = a_{n-5} + a_{n-8} + a_{n-6} + 2b_{n-8} + c_{n-8}\quad (III)an−4​=an−5​+an−8​+an−6​+2bn−8​+cn−8​(III)

Par soustraction de (II)(II)(II) par (III)(III)(III), on obtient pour n⩾8n\geqslant 8n⩾8 :

  • an=−an−8+an−6−an−5+5an−4+an−2+an−1a_{n} = -a_{n-8}+a_{n-6}-a_{n-5}+5a_{n-4}+a_{n-2}+a_{n-1}an​=−an−8​+an−6​−an−5​+5an−4​+an−2​+an−1​

CQFD.

Calculs successifs des premiers termes

On ajoute à la liste des valeurs initiales les nouvelles valeurs avec la relation de récurrence.

Dans notre exemple la valeur suivante est environ le double de la précédente, donc nécessite environ un bit de plus en moyenne. Le stockage de 200 000 valeurs prend donc une place mémoire de plus de 20 milliards de bits ; environ 3 Go.

from time import time
from sys import getsizeof

a = [1, 1, 2, 3, 9, 16, 35, 65] # valeurs initiales
r = [-1, 0, 1, -1, 5, 0, 1, 1] # relation de récurrence
n = 200_000 # rang du terme à calculer

t0 = time()
for _ in range(8, n + 1):
    a.append(sum(ai * ri for ai, ri in zip(a[-8:], r)))
t1 = time()

print(f"Durée : {round(t1 - t0, 2)}s")
print(f"Taille des données : {round(sum(map(getsizeof, a))/10**9, 2)}Go")

m = 10**9 + 7
print(f"Preuve de calcul : {a[-1] % m}")

  • C'est la bonne méthode à employer si on a besoin de (presque) tous les premiers termes d'une suite récurrente linéaire.
  • La dernière instruction affiche le dernier terme calculé, qui est très très grand, modulo un grand nombre premier sur 32bits. Cette preuve de calcul a plusieurs utilités :
    • elle initie au calcul modulaire omniprésent en cryptographie,
    • elle permet de vérifier plus aisément des calculs sur de grands nombres par différentes méthodes,
    • ou par différents langages qui ne travaillent pas avec des nombres plus grands que les mots machine.

C'est à peine plus rapide, mais l'empreinte mémoire est bien plus faible.

Dans un fichier outils.py, créons une fonction rec_lin_suc qui retourne un terme d'une suite récurrente linéaire par calcul des termes successifs.

def rec_lin_suc(a, r, n):
    """Retourne le terme $a_n$ de la suite récurrente linéaire d'ordre $d$ où
    * $a$ est une liste de longueur $d$ : les valeurs initiales ;
    * $r$ est une liste de longueur $d$ : les coefficients de la récurrence ;
    * $n$ est un entier : le range du terme à calculer.
    Méthode avec calculs successifs, complexité O(d×n)
    """
    
    d = len(a) # l'ordre
    assert len(r) == d, "a et r doivent être de même longueur"
    assert n >= 0, "n doit être positif"
    
    if n < d: return a[n]
    a = a[:]
    for _ in range(d, n + 1):
        a.append(sum(ai * ri for ai, ri in zip(a, r)))
        del a[0]
    return a[-1]

Calcul matriciel d'un terme lointain

Dans le code précédent, les deux instructions :

a.append(sum(ai * ri for ai, ri in zip(a, r)))
del a[0]

peuvent être traduites sous forme matricielle par :

a←aMa \leftarrow aMa←aM,

  • où MMM est la matrice compagnon associée au
    polynôme unitaire P(X)=1−X2+X3−5X4−X6−X7+X8P(X) = 1 - X^2 + X^3 -5X^4 - X^6 - X^7 + X^8P(X)=1−X2+X3−5X4−X6−X7+X8 ;
  • et où aaa est un vecteur ligne.
  • On a :
    • P(X)=0  ⟺  X8=−1+X2−X3+5X4+X6+X7P(X) = 0 \iff X^8 = -1 + X^2 - X^3 +5X^4 + X^6 + X^7P(X)=0⟺X8=−1+X2−X3+5X4+X6+X7
    • La dernière colonne de la matrice est notre vecteur de la relation de récurrence linéaire.

De sorte que l'on peut calculer plus rapidement ana_nan​ avec l'exponentiation rapide de matrices.

Si la récurrence est d'ordre ddd,

  • le temps de calcul est en Θ(d3log⁡n)\Theta(d^3 \log n)Θ(d3logn) opérations élémentaires (pas secondes) ;
  • l'empreinte mémoire est en Θ(d2)\Theta(d^2)Θ(d2) coefficients (pas bits) ;
  • Attention, les coefficients peuvent être de grands nombres, et donc le temps de calcul en secondes peut ne pas être en Θ(d3log⁡n)\Theta(d^3 \log n)Θ(d3logn)...

On peut faire mieux en travaillant avec des polynômes.

Calcul polynomial d'un terme lointain

Nous avons pu déterminer un terme de rang nnn d'une suite récurrente linéaire d'ordre ddd en regardant un élément du vecteur calculé avec a×Mna\times M^na×Mn, où aaa est le vecteur des termes initiaux, et MMM la matrice compagnon associée à la récurrence.

Le calcul de MnM^nMn coûte Θ(d3×log⁡n)\Theta(d^3\times \log n)Θ(d3×logn) opérations élémentaires (entre des entiers dont la taille peut devenir très grande...)

On peut réduire sérieusement le coût en considérant que MMM possède PPP comme polynôme minimal, le polynôme associé à la récurrence, et que l'on peut calculer Qn=Xn(modP)Q_n = X^n \pmod PQn​=Xn(modP), par exponentiation rapide ; le coût est Θ(d2×log⁡n)\Theta(d^2\times \log n)Θ(d2×logn). Il suffit d'évaluer QnQ_nQn​ en MMM au lieu de XnX^nXn en MMM afin d'obtenir MnM^nMn.

Mieux encore, on peut se passer totalement de la matrice MMM.

  • On a besoin de la première colonne de MiM^iMi pour i∈[ ⁣[0..d[ ⁣[i\in [\![0..d[\![i∈[[0..d[[,
  • or il s'agit d'un simple vecteur colonne (δi,j)j∈[ ⁣[0..d[ ⁣[(\delta_{i,j})_{j \in [\![0..d[\![}(δi,j​)j∈[[0..d[[​ (0-indexation).

En résumé :

  • Avec Qn=Xn(modP)Q_n = X^n \pmod PQn​=Xn(modP), qui s'écrit : q0+q1X+⋯+qd−1Xd−1q_0 + q_1X + \cdots + q_{d-1}X^{d-1}q0​+q1​X+⋯+qd−1​Xd−1

  • le temps de calcul est en Θ(d2log⁡n)\Theta(d^2 \log n)Θ(d2logn) opérations élémentaires (pas secondes) ;
  • l'empreinte mémoire est en Θ(d2)\Theta(d^2)Θ(d2) coefficients (pas bits) ;
  • Attention, les coefficients peuvent être de grands nombres, et donc le temps de calcul en secondes peut ne pas être en Θ(d2log⁡n)\Theta(d^2 \log n)Θ(d2logn)...

Peut-on faire mieux ?

  • Oui, avec la transformation de Fourier rapide (FFT) qui permet de faire la multiplication de deux polynômes de taille ddd en Θ(dlog⁡d)\Theta(d\log d)Θ(dlogd) au lieu de Θ(d2)\Theta(d^2)Θ(d2). Cela dépasse largement le niveau de papier.
  • Oui, si l'on fait du calcul modulaire en lieu et place du calcul sur les grands entiers. Il y a plusieurs justifications :
    • Posséder plusieurs résidus permet de reconstruire un grand nombre avec le théorème des restes chinois.
    • Le calcul de chaque résidu est parallélisable.
    • Le calcul modulaire est pédagogique, et omniprésent en cryptographie.
    • Le calcul modulaire est possible facilement avec la plupart des langages tant que le modulo tient sur un mot machine. Par contre le calcul sur les grands entiers est plus délicat, en C par exemple.
    • Le calcul modulaire peut faire office de preuve de calcul dans la recherche de nouveaux algorithmes ou les exercices d'algorithmiques.

def rec_lin_suc(a, r, n):
    """Retourne le terme $a_n$ de la suite récurrente linéaire d'ordre $d$ où
    * $a$ est une liste de longueur $d$ : les valeurs initiales ;
    * $r$ est une liste de longueur $d$ : les coefficients de la récurrence ;
    * $n$ est un entier : le range du terme à calculer.
    Méthode avec calculs successifs, complexité O(d×n)
    """
    
    d = len(a) # l'ordre
    assert len(r) == d, "a et r doivent être de même longueur"
    assert n >= 0, "n doit être positif"
    
    if n < d: return a[n]
    a = a[:]
    for _ in range(d, n + 1):
        a.append(sum(ai * ri for ai, ri in zip(a, r)))
        del a[0]
    return a[-1]


def rec_lin_mat(a, r, n):
    """Retourne le terme $a_n$ de la suite récurrente linéaire d'ordre $d$ où
    * $a$ est une liste de longueur $d$ : les valeurs initiales ;
    * $r$ est une liste de longueur $d$ : les coefficients de la récurrence ;
    * $n$ est un entier : le range du terme à calculer.
    Méthode avec matrices, complexité Θ(d^3 log(n))
    """
    
    d = len(a) # l'ordre
    assert len(r) == d, "a et r doivent être de même longueur"
    assert n >= 0, "n doit être positif"

    def matrice_compagnon(r):
        """Retourne la matrice compagnon associée à la récurrence $r$"""
        d = len(r)
        M = [[0]*d for _ in range(d)]
        for i in range(d-1):
            M[i+1][i] = 1
        for i in range(d):
            M[i][-1] = r[i]
        return M

    def puissance(A, n):
        """Retourne la matrice A à la puissance n, où
        * A est une matrice carrée (taille d).
        * n entier positif ou nul.
        Exponentiation rapide, complexité Θ(d^3 log(n))"""

        def produit(A, B):
            """Retourne le produit des deux matrices carrées de taille d : A × B.
            Complexité Θ(d^3)"""
            d = len(A)
            M = [[0]*d for i in range(d)]
            for i in range(d):
                for j in range(d):
                    M[i][j] = sum(A[i][k] * B[k][j] for k in range(d))
            return M
        
        d = len(A)
        M = [[0]*d for i in range(d)]
        for i in range(d):
            M[i][i] = 1
        for digit in bin(n)[2:]:
            M = produit(M, M)
            if digit == "1":
                M = produit(M, A)
        return M

    A = puissance(matrice_compagnon(r), n)
    return sum(a[i] * A[i][0] for i in range(d))


def rec_lin_pol(a, r, n):
    """Retourne le terme $a_n$ de la suite récurrente linéaire d'ordre $d$ où
    * $a$ est une liste de longueur $d$ : les valeurs initiales ;
    * $r$ est une liste de longueur $d$ : les coefficients de la récurrence ;
    * $n$ est un entier : le range du terme à calculer.
    Méthode avec polynômes, complexité Θ(d^2 log(n))
    """
    
    # r = (r_0, r_1, ..., r_{d-1})
    # le calcul polynomial se fait modulo la relation :
    # X^d = r_0 + r_1×X + r_2×X^2 + ... r_{d-1}×X^{d-1}

    d = len(a) # l'ordre
    assert len(r) == d, "a et r doivent être de même longueur"
    assert n >= 0, "n doit être positif"

    def multX(A):
        """Multiplication du polynôme A par X,
        puis réduction modulo la relation.
        Complexité : Θ(d)
        """
        # Multiplication par X
        A.insert(0, 0)
        
        # Réduction
        c_d = A.pop()  # coeff dominant ; celui de X^d
        for i in range(d):
            A[i] += c_d * r[i]
    
    # pré-calcul de X^d à X^{2d-2}, dans liste Xdpi
    # Xdpi := X^{d+i}
    Xdpi =[r[:]]
    for _ in range(d - 2):
        Xdpi.append(Xdpi[-1][:])
        multX(Xdpi[-1])
    
    def produit(A, B):
        """Multiplication des polynômes A et B,
        puis réduction modulo la relation.
        Complexité : Θ(d^2)
        """
        # Multiplication Y := A×B
        Y = [0]*(2*d -1)
        for i in range(d):
            for j in range(d):
                Y[i+j] += A[i] * B[j]
        # Réduction
        for i in range(d - 2, -1, -1):
            ydpi = Y.pop()
            # ydpi est le coeff de X^{d+i} dans Y
            for j in range(d):
                Y[j] += ydpi * Xdpi[i][j]
        return Y

    # Calcul de X^n modulo la relation
    Xn = [0]*d
    Xn[0] = 1 # Xn := X^0
    for digit in bin(n)[2:]:
        Xn = produit(Xn, Xn)
        if digit == "1":
            multX(Xn)

    # Calcul du terme de la suite récurrente linéaire
    return sum(a[i] * Xn[i] for i in range(d))
